# astrbot_plugin_sleep_sense

让 Bot 拥有真实睡眠感知 —— 睡觉、慵懒、熬夜、补觉、吵醒、睡眠周期、做梦，像真人一样。

**版本**: 1.4.0 | **作者**: 夕小柠 | **平台**: AstrBot

---

## 快速开始

### 安装

1. 将插件文件夹放入 AstrBot 的 `data/plugins/` 目录，重启 AstrBot（或在 WebUI 插件页面重载）。
2. 进入 AstrBot WebUI → 插件 → sleep_sense → 打开 `config` 页面，按需调整配置后点击「保存配置」。
3. 所有配置项实时生效，无需重启。

### 依赖

启动时会自动安装 `requirements.txt` 里的依赖（主要是 `Pillow`，用于渲染晚安报告图片）。

## 功能模块

| 模块 | 说明 |
|---|---|
| 慵懒模式 | 到点后回复简短，注入慵懒提示词 |
| 睡觉 | 满足时间+管理员空闲+私聊无消息三条件后入睡，关闭 LLM |
| 起床 | 指定时间自动起床，支持误差波动 |
| 熬夜 | 概率/周限额模式，AI 自主决定，夜晚感性提示词 |
| 熬夜疲劳 | 连续熬夜梯度提示词，第2天起有明显疲劳感 |
| 补觉/午休 | 睡眠不足触发，或每日午休窗口，可配置概率 |
| 吵醒机制 | 按私聊消息数/群聊艾特数触发，两阶段提示词（懵→清醒） |
| 警戒词 | 管理员专属，立刻唤醒 |
| 二次入睡 | 被吵醒后满足条件可重新入睡 |
| 多次吵醒 | 管理员可多次唤醒，提示词风格不同 |
| 睡眠周期 | 一整晚多轮随机浅睡/深睡波动，吵醒难度随阶段变化 |
| 周作息 | 每天可独立配置睡觉/起床时间 |
| 特殊起床 | 上学/早起等场景，指定日期特殊唤醒时间 |
| 周末自主调整 | 熬夜后/周末，AI 可调用工具自行决定晚点起床 |
| 临时闹钟 | AI 工具调用 `sleep_set_alarm` 设置 |
| 噩梦惊醒 | 极低概率触发，情绪不好时概率提高 |
| 被吵醒汇报 | 被别人吵醒后判断是否已互道晚安，决定是否告知管理员 |
| 每日睡眠日志 | 记录每晚入睡/起床时间、阶段切换轨迹、评分、关联梦境 |
| 睡眠评分 | 仿手环算法：时长+深睡占比+连续性+入睡及时性 |
| 晚安报告图片 | 起床后自动生成 Sanrio 风格睡眠报告卡片，4 套配色主题可选 |
| 异地时区 | 支持时区偏移，计算两人作息重叠时间 |
| 屏蔽关键词 | 包含词/前缀词命中后完全不回复，优先级最高 |
| 跨插件事件 | 发出 `sleep_plugin_state_change` 供其他插件监听 |

## 指令

所有指令均为独立扁平指令，仅管理员可用：

```
/睡眠帮助       → 查看完整指令列表
/睡眠状态       → 查看当前状态、上次睡眠时长、熬夜天数
/睡眠周期       → 查看当前所处睡眠周期阶段（浅睡/深睡/正常）
/睡眠昨天       → 查看昨晚完整时间线（入睡/各阶段切换/起床/评分/梦境）
/睡眠统计       → 查看本周睡眠统计
/睡眠报告       → 生成并发送晚安报告图片
/对方           → 查看Bot当前实时睡眠状态（异地恋专用）

/强制清醒       → 强制切换为清醒
/强制慵懒       → 强制切换为慵懒
/强制睡觉       → 强制切换为睡眠
/强制熬夜       → 强制切换为熬夜
/强制午休       → 强制切换为午休/补觉

/测试做梦       → 立刻生成一次梦境（测试用）
/测试起床       → 触发起床流程（测试用）
/测试噩梦       → 触发噩梦惊醒提示词（测试用）
/测试闹钟       → 查看当前临时闹钟列表

/睡眠日志开     → 开启 debug 日志
/睡眠日志关     → 关闭 debug 日志

/梦境列表       → 查看最近5条梦境记录
/梦境今晚       → 查看今晚的梦境详情
/梦境清除       → 清除全部梦境记录
```

也支持机器人昵称唤醒方式触发，例如「渊渊 睡眠状态」。

## 文件结构

```
data/sleep_sense/
├── state.json            状态（自动维护）
├── alarms.json           临时闹钟
├── dreams.json           梦境记录
├── daily_log.json        每日睡眠日志（阶段轨迹、评分）
├── report_*.png          生成的晚安报告图片
├── logs/
│   └── sleep.log
└── stats/                统计数据（按周存档）
    └── 2026-W23.json
```

配置统一通过 AstrBot WebUI 的插件配置页面（`_conf_schema.json` 驱动）管理。

## 跨插件联动

其他插件可以监听睡眠状态变化事件 `sleep_plugin_state_change`，状态值为 `sleep`/`awake`/`nap`/`overtime` 等，可用于联动关闭/开启主动回复逻辑。

---

## WebUI 配置说明

AstrBot WebUI → 插件 → sleep_sense → config 页面，所有配置项均有中文说明，修改后点击「保存配置」即时生效，无需重启。

晚安报告图片支持 4 套配色主题（蓝白/粉白/黄白/紫白）以及单独的模块开关（时长/周期轨迹/评分/梦境/重叠时间），均在场景开关页面配置。

---

## 扩展开发

- 新增指令请用独立的 `@filter.command("指令名")` 注册一步到位的扁平指令，避免用 `command_group` 拼接子命令，在部分唤醒词触发场景下子命令解析不稳定。
- 新增功能请在 `main.py` 中添加对应的 `_handle_*` 或 `_check_*` 方法。
- 所有配置项在 `_conf_schema.json` 中定义，WebUI 自动生成对应输入框。
